🗃️ Data Model — Payload Collections

9 collections sesuai PRD, dengan relasi lengkap

Entity Relationship
Users ──┬──> Authors (link via `user` field)
        └── createdBy/updatedBy pada Posts, Media, dll

Authors ──> Posts.author (relation hasOne)
Categories ──> Posts.category (relation hasOne)
           └── Categories.parent (relation hasOne — self)
Tags ──> Posts.tags (relation hasMany)
Media ──> Posts.featuredImage, Posts.ogImage, Authors.avatar, Settings.logo, dll

Posts ──> Redirects (auto-create saat slug berubah)
Pages ──> Redirects (auto-create saat slug berubah)

Settings — singleton, global config
Redirects — standalone, no relasi ke collection lain

👤 Users

Internal auth via Better Auth

name, email, role (admin/editor/author/viewer)
⚡ Better Auth handle login session

✍️ Authors

Public author profile

name, slug, bio, avatar, email, socialLinks
Bisa terhubung ke User (opsional)

📄 Posts

Core — artikel berita

title, slug, excerpt, content (richText), featuredImage
category, tags[], author, status (draft→review→published→archived)
isFeatured, isBreakingNews, seoTitle, seoDescription
⚡ Slug change → auto redirect 301

📂 Categories

Hierarki kategori

name, slug, description, parent, order, isActive
Nasional, Madura, Politik, Ekonomi, dll

🏷️ Tags

Tags/label artikel

name, slug, description, isActive

🖼️ Media

Upload gambar

file, alt (wajib), caption, credit
Image sizes: thumbnail(400x300), card(768x432), hero(1280x720), og(1200x630)

📃 Pages

Halaman statis

title, slug, content, seoTitle, seoDescription, status

🔀 Redirects

301/302 manager

from, to, statusCode (301/302), isActive
Auto-create saat slug berubah
