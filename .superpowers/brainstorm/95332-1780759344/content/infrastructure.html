⚡ Infrastructure (Phase 4)

Meilisearch + Redis + Docker Deploy + R2/S3 Ready

🔍 Meilisearch

Index: posts
Sync via Payload hooks (afterChange, afterDelete)
Masuk index: status=published & publishedAt≤now & allowIndex=true
Keluar index: archived, deleted, unpublished
Search page consume Meilisearch API langsung

⚡ Redis Cache

Cache keys:
homepage, category:{slug}:page:{n},
tag:{slug}:page:{n}, post:{slug}, popular-posts
Invalidate: via hooks saat publish/edit/delete
View counter: views:post:{id} → sync ke DB

🐳 Docker Compose

4 services: app (Next.js), postgres, redis, meilisearch
Volume persistent untuk media
Env vars via .env
Health checks tiap service

Image component terpusat

🔗 Hook System — Sync Pipeline

Payload Hooks
Posts.afterChange ──┬──→ Meilisearch sync (publish/unpublish)
                    ├──→ Redis cache invalidation
                    ├──→ Redirect auto-create (slug change)
                    └──→ Regenerate sitemap (future)

Posts.beforeDelete ──→ Meilisearch remove
                     → Redis cache invalidation

Categories.afterChange ──→ Redis cache invalidation
Tags.afterChange ──→ Redis cache invalidation
Settings.afterChange ──→ Redis cache invalidation
